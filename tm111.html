Evidence

Short description of what the program does and how it works

When the green flag is clicked, the program asks the user to calculate the price increase between an old price and a new price. The user’s answer is stored in a variable and compared against the correct calculated value. If the answer is correct, the program displays confirmation feedback. If the answer is incorrect, the program displays the correct price increase.

Problem statement

When learning programming, it can be difficult to understand how numerical data, user input, and conditional logic work together in a single program. Without clear examples, learners may struggle to see how variables store values and how decisions are made based on comparisons. This program demonstrates how user input can be processed, checked, and responded to using variables and selection.

The idea

The idea was to create an interactive Scratch program that asks the user a question involving price changes, calculates the correct answer internally, and then checks the user’s response to provide immediate feedback.

Brief reflection

This idea was stronger than others because it clearly demonstrates multiple programming concepts working together, including variables, calculations, user input, and conditional logic. The program also produces clear feedback, making it easy to test, debug, and explain.

Overview

A series of structured programming tasks completed as part of TM111, focused on understanding program logic, data handling, and problem decomposition.

What I did

  • Designed and built small programs using Scratch
  • Implemented algorithms using sequence, selection, and iteration
  • Used variables and lists to store and process data
  • Tested and debugged programs to ensure correct behaviour

Tools and technologies

Scratch, algorithm design, computational thinking

What I learned

This project strengthened my understanding of how programs process data step by step. I learned how to translate problems into logical instructions and how to test and refine solutions when errors occurred.

Evidence

Short description of how the program works

When the green flag is clicked, the program asks the user to calculate the price increase between an old price and a new price. The user’s answer is stored in a variable and compared against the correct value. Feedback is then displayed based on the result.

Problem statement

Beginners often struggle to understand how variables, user input, and conditional logic work together. This program provides a clear example of storing data, checking values, and responding logically.

The idea

The idea was to create an interactive Scratch program that asks the user a question involving price changes, calculates the correct answer internally, and then checks the user’s response to provide immediate feedback.

Brief reflection

This idea was strong because it clearly demonstrates multiple programming concepts working together, including variables, calculations, user input, and conditional logic. It also produces clear feedback, which makes it easier to test, debug, and explain.
